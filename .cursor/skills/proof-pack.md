# Skill: proof-pack

## Purpose

Make every implementation produce evidence.

## Use when

- Building the current slice.
- Fixing a bug.
- Changing logic.
- Touching auth, payments, AI, or backend behavior.

## Required proof

- Files changed
- Commands run
- Test or build result
- Screenshot, log, or output if relevant
- Pass or fail status
- Remaining unknowns

## Rules

- No merge-ready claim without evidence.
- If proof was not run, label the result as unverified.
- Prefer the smallest proof that validates the target behavior.
