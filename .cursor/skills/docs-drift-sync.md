# Skill: docs-drift-sync

## Purpose

Keep docs aligned with actual implementation.

## Use when

- Implementation changes scope.
- Architecture meaningfully changes.
- Proof slice changes.
- Backlog order changes.
- Risk status changes.

## Files to check

- `docs/APP_SCOPE.md`
- `docs/ARCHITECTURE.md`
- `docs/BUILD_SHEET.md`
- `docs/NEXT_EXECUTION_BACKLOG.md`
- `docs/HARD_RISKS.md`

## Rules

- Update only affected docs.
- Do not rewrite everything.
- Keep docs concise.
- Reflect actual repo truth, not aspirational plans.

## Output

- Docs updated
- Why
- What changed in repo that triggered the update
