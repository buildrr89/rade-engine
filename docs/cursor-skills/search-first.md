# Skill: search-first

## Purpose

Force repo and platform search before invention.

## Use when

- Proposing a new dependency.
- Proposing a new helper or abstraction.
- Proposing a new pattern or library.
- Unsure whether the repo already contains a solution.

## Rules

- Search the repo for existing implementation first.
- Search `pyproject.toml`, package manifests, and lockfiles first.
- Search existing components and utilities first.
- Only propose new additions if the current repo cannot support the need.

## Output

- Existing matches found
- Reuse option
- Gap identified
- Minimal justified addition
